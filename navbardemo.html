Try #3

FAIL

Now we want to adjust the width of the buttons and put some horizontal space between them. All of this cannot be done! We can only put the space between the buttons. Why? According to the CSS specification:

Inline elements: The width is as long as the text or the containing image, and cannot be manipulated. Further, height, line-height, and top and bottom margins/padding cannot be manipulated. However, left and right margins/padding can be manipulated. By default, <a> is an inline element.

Block elements: The width of a block element is by default 100% of the containing element unless a width is specified. Futher, height, line-height, and top and bottom margins/padding can be maniuplated. By default, <ul> and <li> are block elements.

Note that the margins are shown in red, i.e. the color of the item behind the list item, since the list item margin is transparent.

    ul {
        list-style: none;
        height: 70px;
    }

    li {
        display: inline;
        height: 60px;
        width: 160px;
        margin-right: 20px;
    }

Try #4

FAIL

Similarly, we'll try to adjust the height and width of the buttons using the anchor tag instead of the list tag. Because the anchor tag is an inline element by default, this approach fails for the same reasons as above.

Note that the margin area to the right of the button is shown in green, the color of the list item rather than the color of the anchor item which actually has the margin. This is because a margin is transparent and therefore shows the margin in the color of the thing behind it (the list item). In contrast, padding is not transparent. Try changing margin-right to padding-right and note that the margin color changes to blue, the color of the anchor.

    ul {
        list-style: none;
        height: 70px;
    }

    li {
        display: inline;
    }

    a {
        height: 60px;
        width: 160px;
        margin-right: 20px;  // Try padding-right: 20px instead!
    }

Try #5

FAIL

The previous failures in adjusting the height and width of the navigation buttons relate to the use of inline elements, namely the anchor tag which is inline by default, and by declaring the list item tag as inline. Instead of using inline list items to cause the buttons to be arranged horizontally, we'll use float: left. The question then becomes, "what gets floated, the list item or the anchor?" First, try the list item as shown below

This method provides the desired horizontal arrangement of the buttons, but the width and height specifiers are placed on the anchor tag which is still an inline element. The size of the buttons is not correct.

    ul {
        list-style: none;
        height: 70px;
    }

    li {
        float: left;
    }

    a {

Try #7

SUCCEED (Mysterious)

Alternatively, float the anchor element. This appears to automatically turn the anchor from a default inline element into a block element. Mysteriously, the buttons appear horizontally even though the list items aren't floated. Why does this work?

    ul {
        list-style: none;
        height: 70px;
    }

    a {
        float: left;

Try #8

SUCCEED

One way to solve the problem is to continue floating the list item, and specifying the size of the anchor element along with explicitly making the anchor into a block item.

    ul {
        list-style: none;
        height: 70px;
    }

    li {
        float: left;
    }

    a {
        display: block;
        height: 60px;
        width: 160px;
        margin-right: 20px;
    }
